网络与状态机的构想

如何让 apple app store 绕过 “用户登入校验” 直接进行 app 升级？
服务器端准备好了 app 新版本的二进制软件, 并且准备好了让它作为 HTTP 响应报文 发回给客户端
客户端发送请求报文, 整个 HTTP 请求报文 携带的是 客户端的 “状态”

在 `网络与状态机的构想` 里
服务器端 阅读这个状态 (由 HTTP 请求报文携带的)，然后同意给客户端发，就发了(带有二进制软件的 HTTP 响应报文)

错误状态后果: 服务器端拒绝发送 (带有二进制软件的 HTTP 响应报文) // 这是 “你” 想要的. 然后你被提示该如何才能获得它
正确状态里可能携带的:
- app 相关 (app 名, app id, app 版本请求 比如 latest LTS)
- 过卡相关 (iOS 版本, ) <- 这里发生了 “用户登入校验”
- ... // 总之 你的接收方状态，就体现在 发送的报文 里。响应方也是看 发送的报文 (是否‘合格’) 才发放 ‘奖品’。对 是否‘合格’的判断是从 route+controller+service 开始的, 如果要绕过 就可以从 (修改版)service 绕过。最后客户端得到了 ‘奖品’ 反而说明了它之前发送的状态(给服务器端读到了)是没问题的
