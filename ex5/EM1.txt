
对 event machine 的合理性观察
web backend app 是基于 event driven model 的一个运用
- 几个状态机
- 几个状态机如何交流的

那么 第一个状态机是 根据外部条件1 改变自己的内部状态 最后 产生 effect , 这里可以用 if else 也可以表驱动法
// 外部条件1: HTTP 报文, 内部状态: , 产生 effect: 根据内部状态 (读取某值) 对外产生 effect (保存到假设的地点, 然后由谁读取)

第二个状态机是 根据外部条件2 改变自己的内部状态 最后 产生 effect
// 外部条件2: 状态机读取器要读取的值, 内部状态: , 产生 effect: 读取内部状态 对外产生 effect (保存到假设的地点, 方便读取)
